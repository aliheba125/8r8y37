# 🩺 Shadow VIP — نسخة التشخيص (Diagnostic Build)

> **الغرض:** هذه النسخة **لا تُصلح** أي شيء. وظيفتها الوحيدة **تسجيل كل خطوة** أثناء محاولة تسجيل الدخول (Facebook / Twitter / Google) داخل PUBG، حتى نعرف **بالدليل الحي** أين تتوقف العملية بالضبط — بدل التخمين.

الملف: `Shadow_VIP_DIAG.apk`

---

## ✅ ماذا تسجّل هذه النسخة؟

كل حدث يُطبع في logcat تحت الوسم (tag): **`PUBGDIAG`**، وبهذا الشكل:

```
#<رقم تسلسلي> [<الوقت HH:mm:ss.SSS>] S<رقم الجلسة> <الحدث> <التفاصيل>
```

| الوسم في السجل | معناه |
|---|---|
| `STARTACT act=... dat=... cmp=... pkg=...` | كل `startActivity` مع الرابط الكامل الذي يحاول فتحه |
| `BLOCK_HTTP_VIEW_RETURN_-1 ... dat=https://...` | 🎯 **الدليل الحاسم**: وصل رابط ويب وتم حجبه بـ `return -1` |
| `FBHOOK_beforeHook_FIRED arg[0]=...` | هل عمل FacebookLoginHook فعلاً (يحسم مسألة "الكود الميت") |
| `WEBVIEWHOOK_beforeHook_FIRED arg[1]=...` | هل عمل WebViewHook فعلاً |
| `BINDSVC act=... pkg=...` | ربط خدمة (مثل CustomTabs بالمتصفح) |
| `GETPKGINFO pkg=...` | استعلام عن حزمة facebook/twitter/gms (فحص التوقيع/التوفّر) |
| `======== LOGIN SESSION #N START :: ... ========` | بداية محاولة تسجيل دخول جديدة (يُرقّم تلقائياً) |
| `... EX= <StackTrace>` | أي استثناء مع أثر المكدس الكامل |

**ميزات احترافية مضمّنة:** توقيت زمني لكل سطر ✅ — رقم جلسة لكل محاولة ✅ — StackTrace كامل للأخطاء ✅ — حفظ في ملف ✅

---

## 🔴 حدود صادقة (اقرأها)

- تسجّل **كل ما يخرج من PUBG إلى النظام** (Intent/الحزمة/الخدمة). وهذا غالباً يكفي لمعرفة السبب.
- **لا تستطيع** رؤية ما يجري **داخل** كود Facebook/Twitter SDK نفسه (بدء OAuth، إنشاء WebView داخلي) لأن هذا داخل PUBG المحجوب. نراه فقط بشكل غير مباشر عبر ما يستدعيه.

---

## 📲 خطوات الاختبار

### 1) التثبيت
- احذف النسخة الحالية، وثبّت `Shadow_VIP_DIAG.apk`.
- سجّل الدخول للتطبيق طبيعياً (مفتاح: `ali2026` أو `vip` … إلخ).

### 2) التقاط السجل — الطريقة الأدق (كمبيوتر + adb)
هذه تعطيك **ملفاً واحداً بالتوقيت** جاهزاً للإرسال:
```bash
adb logcat -c                                   # مسح السجل القديم
adb logcat -v time PUBGDIAG:V *:S > pubg_login.txt
```
اترك هذا الأمر يعمل، ثم:
1. افتح PUBG من داخل التطبيق.
2. اذهب لتسجيل الدخول، اضغط **Facebook** (جرّب مع تطبيق فيسبوك مثبّت، ثم بدون).
3. ارجع واضغط **Twitter**.
4. أوقف الأمر (Ctrl+C) وأرسل لي ملف `pubg_login.txt`.

### 3) بديل بدون كمبيوتر (ملف داخل الجهاز)
النسخة تكتب أيضاً — أفضل جهد — ملفاً في:
```
/sdcard/Android/data/<حزمة اللعبة>/files/pubg_login_diag.log
```
> ملاحظة: بسبب إعادة توجيه الملفات في البيئة الافتراضية، قد يُعاد توجيه المسار أو يتعذّر الوصول على بعض الأجهزة (Android 11+). لذلك **الطريقة عبر adb أضمن**. ملف الجهاز لا يبدأ بالكتابة إلا بعد **أول محاولة تسجيل دخول** (لتقليل الضوضاء).

---

## 🔎 كيف نقرأ النتيجة؟

- **لو ظهر** `BLOCK_HTTP_VIEW_RETURN_-1 dat=https://...facebook/twitter/oauth...` → **إثبات قاطع** أن الحجب هو السبب.
- **لو لم يظهر أي `STARTACT` بـ VIEW/https** أثناء الدخول → PUBG يستخدم مساراً آخر (WebView داخلي / SDK أصلي) والحجب **ليس** السبب.
- **لو ظهر `FBHOOK_beforeHook_FIRED`** → أي أن الـhook ليس ميتاً (يخالف تحليلي الساكن). لو لم يظهر أبداً → يؤكد أنه ميت.
- **`GETPKGINFO pkg=com.facebook.katana`** يظهر متى فحص PUBG وجود تطبيق فيسبوك.

أرسل لي الملف وسأحلّله سطراً سطراً وأحدّد نقطة التوقف الحقيقية بالدليل.
